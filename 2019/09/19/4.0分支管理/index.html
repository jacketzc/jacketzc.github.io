4.0分支管理

新建一个分支
git branch dev（新分支的名字）
切换分支
git checkout dev
或者git switch dev
查看当前工作分区
git branch
//新建与切换分支的命令可以简化为git checkout -b dev（或者git switch -c dev）

master分支是一条时间线，新分支是从原master分支最新节点开辟出的新时间线，两条时间线分别工作，互不影响
//分支的时间线实际上是由一个指针控制的，如master分支拥有一个HEAD指针，而创建新分支等于新建了一个指针指向新的分支
//dev分支可以独立add、commit，尽管操作的可能是同一个文件，当切换分支时，文件的内容是不同的，文件的内容依据当前使用分支的最新commit点

合并分支
git merge dev
//将指定分支与当前分支合并
//此次提交为fast-forward模式（因为合并的逻辑非常明显）

删除分支
git branch -d dev
//git提倡新开辟一个分支以取代直接在master分支上工作，这样更安全

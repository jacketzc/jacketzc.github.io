19.00File类

文件和目录路径名的抽象表示形式
//该类可以应用于任何的系统

成员变量（全部都是静态的）
//这是java为跨平台做出的设计，以后用到分隔符的地方尽量使用该成员变量
string pathSeparator
//路径分隔符 windows（；） linux（：）
string sepatator
//名称分隔符 windows（\） linux（/）

绝对路径和相对路径
绝对：从盘符或者根目录开始
相对：以当前目录为基准
//当前目录为c:\user，那么该目录下的文件夹可以直接写作123.txt
注意：
java使用两个‘\’代表一个普通的反斜杠

构造方法
File(String filename)
//路径可以是文件，也可以是文件夹
//可以使绝对路径，也可以是相对路径
//路径可以存在，也可以不存在
//不区分大小写
File(String parent,String child)
//将路径分割为两部分，便于灵活使用
//"c:\\"(parent)和"123.txt"可以组装成一个完整的路径
File(File parent,String child)
//父路径使用一个File对象
//还有一种不规范的写法File(File+"\\chile")，因为是字符串拼接，所有中间的"\\"不可以省略

基本方法：
.getAbsolutePath()
//获取构造方法中传递路径的绝对路径
.getPath()
//获取构造方法中传递的路径
//toString方法调用的就是这个方法
.getName()
//获取构造方法传递路径的结尾
//这个结尾往往是一个文件夹或者文件名
.getLength()
//返回文件大小（以字节为单位）
//文件夹或者不存在的地址返回0

判断方法
bool exists()
//判断构参中的路径是否存在
bool isDirectory()
//判断是否是一个文件夹
bool isFile()
//判断是否是一个文件

增删方法
bool createNewFile()
//创建此File表示的文件（如果不存在）
//文件以及存在或者文件父级目录不存在都会抛出异常
//此方法不能创建文件夹（以路径创建仍然创建的是一个文件）
bool delete()
//删除此File表示的文件或者目录
//此方法无法删除一个非空的目录
//此方法删除的文件不会被回收且不可撤销！
bool mkdir()
//创建此File表示的单级文件夹
//也就是说，不能创建该文件夹时同时创建它的父级文件夹

遍历方法
String[] list()
//返回该路径下所有的直接子目录或者文件所表示的String数组
//没有任何的子目录会返回一个空数组
//数组成员为文件或者文件夹的名称
//子目录的子目录不会被遍历
//路径不存在或者不是路径都会抛出空指针异常
File[] listFiles()
//返回File数组
//成员为文件或者文件夹的绝对路径

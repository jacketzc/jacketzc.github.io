1.17设计思想

## 控制层永不处理业务逻辑
  • 控制层只负责接受请求、然后调用业务层的方法、然后返回
  • 任何与业务逻辑相关的代码应该下放到业务层（哪怕是1+1）

## 程序永远可能出错！
  • 在控制层使用最大的Exception包裹需要处理的业务代码，如果一切顺利，返回success的返回类型，如果发生异常，返回相应的自定义错误代码
  • 示例：
public ReturnResult generateOrder( @RequestBody OrderInfoParam param) {
    int code = 0;
    try {
        code = service.generateOrder(param);
        if (code == 0) {
            return ReturnUtil.returnSuccess();
        }else
            return ReturnUtil.returnFail(code);
    } catch (Exception e) {
        e.printStackTrace();
        return ReturnUtil.returnFail(555);
    }
}


## 快的任务同步做，慢的任务异步做
  • 往redis中存数据同步完成即可
  • 往mysql中存数据异步完成即可

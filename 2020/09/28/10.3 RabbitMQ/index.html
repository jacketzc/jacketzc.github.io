10.3 RabbitMQ

## 说明
  • rabbitmq的队列是通过@Configure事先配置好的，而不像activeMQ那样指定Destination
  • rabbitmq在普通消息队列的基础上，引入了交换机的概念，发送者的消息全部发送给交换机，由交换机来调度消息
    • 绑定的规则被称为路由键（ routing key）
  • 管理页面的地址为：http://localhost:15672/

## 注意
  • 可以手动创建queue或者用代码生成queue，两者效果一致
    • 代码中使用@Bean并且返回Queue的方法即为生成一个queue
    • 生成的queue不是一次性的（只要rabbitmq还开着，java程序关掉这个queue仍然存在）！如果需要修改其中的属性，需要去管理页面删掉原来的queue

## 依赖
<!--rabbitmq-->
<dependency>
   <groupId>org.springframework.boot</groupId>
   <artifactId>spring-boot-starter-amqp</artifactId>
   <scope>provided</scope>
</dependency>
<!--rabbitmq-->

## 配置
#rabbitmq
spring.rabbitmq.host=localhost
spring.rabbitmq.port=5672
spring.rabbitmq.username=guest
spring.rabbitmq.password=guest

# 说明
## direct模式
  • exchange通过binding_key绑定一个queue
    • 一个exchange可以绑定多个queue
  • 发送消息时，该消息携带：要发往的exchange、routing_key、msg
  • 对应的exchange接受到该消息，将消息推送到使用与routing_key完全一致的binding_key绑定到该exchange的queue
    • 如果binding_key为dog，那么routing_key必须为dog，dog.teddy不行

## fanout模式
  • 发向该模式下exchange的消息，会发向所有绑定到该exchange的queue中
  • 相当于广播了一条消息

## topic模式
  • 相当于在direct模式的基础上，添加通配符匹配
    • “#”表示匹配0或者多个单词，“*”表示匹配一个单词
  • 消息将发送给所有匹配到的队列
    • 每个队列都会收到消息哦
  • 这个模式下binding_key和routing_key的字符串被切分为单词，这些单词使用“.”隔开
    • 如dog.teddy

## 配置类
  • 所有exchange、queue、binding都在该配置类中配置
    • 构建queue可以指定一些入参作为属性
    • public Queue(String name, boolean durable, boolean exclusive, boolean autoDelete, Map<String, Object> arguments)
      • name：该队列的名称
      • durable：是否持久化，将决定该队列重启后是否还存在
      • exclusive：是否独有，如果是，该队列只有队列的创建者能使用
      • autodelete：是否在该队列不再使用时自动删除
      • arguments：构建该队列时所添加的参数（比如指定队内消息的过期时间）
@Configuration
public class RabbitMQConfigure {
    //定义几个路由名称
    public final static String DIRECT_EXCHANGE = "exchange.direct";
    public final static String FANOUT_EXCHANGE = "fanout.direct";
    public final static String TOPIC_EXCHANGE = "topic.direct";
    //定义几个binding_key
    public final static String DIRECT_BINDING_KEY = "direct.message";
    public final static String TOPIC_BINDING_KEY_A = "topic.message";
    public final static String TOPIC_BINDING_KEY_B = "topic.#";
    //定义几个queue
    public final static String QUEUE_A = "queue.a";
    public final static String QUEUE_B = "queue.b";
    public final static String QUEUE_C = "queue.c";
    /**
    * 自定义消息转换器（可选）
    * 使用 json转换器替代jdk默认的序列化转换器
    * @return
    */
    @Bean
    public MessageConverter messageConverter() {
        return new Jackson2JsonMessageConverter();
    }
    //几个queue，构建queue的时候，可以根据具体情况去配置相应的属性
    @Bean
    public Queue queueA() {
        return new Queue(QUEUE_A);
    }
    @Bean
    public Queue queueB() {
        return new Queue(QUEUE_B);
    }
    @Bean
    public Queue queueC() {
        return new Queue(QUEUE_C);
    }
}
## utils
  • 发送不同消息的方法将写在这里
  • 发送方法依赖于rabbitTemplate
@Component
public class RabbitMQUtils {
    @Autowired
    private RabbitTemplate rabbitTemplate;
}


# 实验
## direct模式
  • 配置一个direct交换机
@Bean
public DirectExchange directExchange() {
    return new DirectExchange(DIRECT_EXCHANGE);
}
  • 将这个交换机绑定到queue.a
@Bean
public Binding bindingDirect() {
    return BindingBuilder.bind(queueA()).to(directExchange()).with(DIRECT_BINDING_KEY);
}
  • 添加发送direct消息的方法（在utils中）
public void sendDirectMsg(Object msg) {
    rabbitTemplate.convertAndSend(DIRECT_EXCHANGE, DIRECT_BINDING_KEY, msg);
}
  • 测试（在controller中）
    • 设置@RabbitListener表明方法监听哪个队列发送过来的消息
      • 该注解可以用于类或者方法
    • 为方法添加@RabbitHandler，表明该方法将处理队列中的消息
      • 只能用于方法
@RestController
@RabbitListener(queues = QUEUE_A)
@RequestMapping("/rabbit")
public class RabbitMQController {
    @Resource
    private RabbitMQUtils rabbitMQUtils;

    @GetMapping("sendDirectMsg")
    public void sendDirectMsg() {
        EmailInfo emailInfo = new EmailInfo();
        emailInfo.setTitle("hello");
        emailInfo.setSendTo("jacketzc");
        rabbitMQUtils.sendDirectMsg(emailInfo);
    }
    @RabbitHandler
    public void getDirectMsg(Object msg) {
        log.info(msg.toString());
    }
}

## fanout模式
  • 配置一个fanout交换机
@Bean
public FanoutExchange fanoutExchange() {
    return new FanoutExchange(FANOUT_EXCHANGE);
}
  • 将三个队列都绑定到这个exchange
@Bean
public Binding bindingFanoutA() {
    return BindingBuilder.bind(queueA()).to(fanoutExchange());
}
@Bean
public Binding bindingFanoutB() {
    return BindingBuilder.bind(queueB()).to(fanoutExchange());
}
@Bean
public Binding bindingFanoutC() {
    return BindingBuilder.bind(queueC()).to(fanoutExchange());
}
  • 添加发送fanout消息的方法（utils）
    • 虽然fanout消息会忽略掉routing_key，但是这个routing_key必须加
public void sendFanoutMsg(Object msg){
    rabbitTemplate.convertAndSend(FANOUT_EXCHANGE,"", msg);
}
  • 测试（controller）
    • 发送的消息三个队列都会收到
@GetMapping("sendFanoutMsg")
public void sendFanoutMsg() {
    EmailInfo emailInfo = new EmailInfo();
    emailInfo.setTitle("hello");
    emailInfo.setSendTo("jacketzc");
    rabbitMQUtils.sendFanoutMsg(emailInfo);
}

## topic模式
  • 配置一个topic交换机
@Bean
public TopicExchange topicExchange() {
    return new TopicExchange(TOPIC_EXCHANGE);
}
  • 将topic交换机绑定到两个队列上，其中的一个binding_key将使用通配符
@Bean
//这个binding_key为topic.a
public Binding bindingTopicA() {
    return BindingBuilder.bind(queueA()).to(topicExchange()).with(TOPIC_BINDING_KEY_A);
}
@Bean
//这个binding_key为topic.#
public Binding bindingTopicAll() {
    return BindingBuilder.bind(queueB()).to(topicExchange()).with(TOPIC_BINDING_KEY_B);
}
  • 创建发送topic消息的方法（utils）
//该方法发送的routing_key（topic.a），将被 topic.a以及topic.# 同时匹配
public void sendTopicMsgAll(Object msg) {
    rabbitTemplate.convertAndSend(TOPIC_EXCHANGE, TOPIC_BINDING_KEY_A, msg);
}
//该方法发送的该方法发送的routing_key（topic.b），只能被topic.# 匹配
public void sendTopicMsgA(Object msg){
    rabbitTemplate.convertAndSend(TOPIC_EXCHANGE, TOPIC_BINDING_KEY_B, msg);
}
  • 测试（controller）
    • sendTopicMsgA发送的消息只能被queueB接收（routing_key为“topic.b”，只有queueB的binding_key为“topic.#”能匹配）
    • sendTopicMsgAll发送的消息将会被queueA和B同时接受
@GetMapping("/sendTopicMsgA")
public void sendTopicMsgA() {
    EmailInfo emailInfo = new EmailInfo();
    emailInfo.setTitle("hello");
    emailInfo.setSendTo("jacketzc");
    rabbitMQUtils.sendTopicMsgA(emailInfo);
}
@GetMapping("/sendTopicMsgAll")
public void sendTopicMsgAll() {
    EmailInfo emailInfo = new EmailInfo();
    emailInfo.setTitle("hello");
    emailInfo.setSendTo("jacketzc");
    rabbitMQUtils.sendTopicMsgAll(emailInfo);
}

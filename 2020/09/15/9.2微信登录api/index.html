9.2微信登录api

## 说明
如果用户在微信客户端中访问第三方网页，公众号可以通过微信网页授权机制，来获取用户基本信息，进而实现业务逻辑。
//只有在微信客户端中，才能知道openid

## 参考
  • 微信开放平台
https://mp.weixin.qq.com/debug/cgi-bin/sandbox?t=sandbox/login
  • 文档   
https://developers.weixin.qq.com/doc/offiaccount/OA_Web_Apps/Wechat_webpage_authorization.html#1

## 流程
  • 向微信登录接口发送请求，用户同意后可以获取code
    • 该请求的回调方法为第二步，第二步与第三步的方法写在一起
  • 使用code向微信登录接口发送请求，获取access_token以及用户的openid
    • openid就是加密后的微信号
  • 根据access_token和openid换取用户信息

## 步骤
  • 1.创建授权请求链接，用户同意授权后，将获取code
    • 参考以下链接来构建获取code的链接：https://open.weixin.qq.com/connect/oauth2/authorize?appid=wxf0e81c3bee622d60&redirect_uri=http%3A%2F%2Fnba.bluewebgame.com%2Foauth_response.php&response_type=code&scope=snsapi_userinfo&state=STATE#wechat_redirect
    • 授权链接的构成：
      • appid：公众号的唯一标识
      • redirect_uri：授权后重定向的回调链接地址
        • 此地址此处由ngrok将本地服务内网穿透后放到公网域名生成的地址
          • 使用ngrok http 8082 命令将localhost:8082绑定到http://c5654eba014e.ngrok.io
        • 在微信开发平台填写认证的时候，将此链接去掉http：也就是c5654eba014e.ngrok.io（要求填写的是回调域名，不是链接！）
      • response_type：返回类型，请填写code
        • 此处为固定的，也就是：response_type=code
      • scope：应用授权作用域
        • snsapi_base：不弹出授权页面，直接跳转，只能获取用户openid
        • snsapi_userinfo：弹出授权页面，可通过openid拿到昵称、性别、所在地（并且， 即使在未关注的情况下，只要用户授权，也能获取其信息）
      • state：（非必填）重定向后会带上state参数，可以验证链接的来源
        • 向vx的api使用该链接发送请求时添加该参数后，vx服务器在返回的链接中也会带上该参数
      • #wechat_redirect：无论直接打开还是做页面302重定向时候，必须带此参数
    • 创建请求链接：
      • 将链接中固定的部分保存到application.properties
wx.login.appID=wxcb123a4dd8a20ac5
wx.login.redirectUri=https://c5654eba014e.ngrok.io/vx/getUserInfo
wx.login.codeUri=https://open.weixin.qq.com/connect/oauth2/authorize?
      • 创建WxConfig，专门用于生成该连接：
@Data
@Component
@ConfigurationProperties("wx.login")
public class WxConfigure {
    private String appID;
    private String redirectUri;
    private String codeUri;
    public String getCode() {
        StringBuffer stringBuffer = new StringBuffer(getCodeUri());
        stringBuffer.append("appid=").append(appID);
        stringBuffer.append("&redirect_uri=").append(redirectUri);
        stringBuffer.append("&response_type=code");
        stringBuffer.append("&scope=snsapi_userinfo");
        stringBuffer.append("#wechat_redirect");
        return stringBuffer.toString();
    }
}
    • 创建获取code的请求接口，该接口将请求重定向到微信的获取code的接口
@GetMapping(value = "/getCode")
public String getCode() {
    //此处要重定向，所以应该使用@Controller而不是@RestController
    return "redirect:" + wxConfigure.getCode();
}
    • 创建回调接口接受code
      • 该回调接口是写在请求链接的redirect_uri的
@RequestMapping("/getUserInfo")
public void getUserInfo(@RequestParam String code) {
    log.info(code);
}
  • 2.通过code换取网页授权access_token
    • //access_token一段时间后会过期
    • 参考以下链接来构建获取access_token以及openid的链接： https://api.weixin.qq.com/sns/oauth2/access_token?appid=APPID&secret=SECRET&code=CODE&grant_type=authorization_code
      • 链接构成：
        • appid
        • secret：公众号的appsecret
        • code：填写第一步获取的code参数
        • grant_type：填写为authorization_code（固定）
    • 将固定写死的部分放入application.propreties
wx.login.accessTokenUrl=https://api.weixin.qq.com/sns/oauth2/access_token?
wx.login.secret=aed33f0cb6bea8d0e9defeb419e85870
    • 构建通过code获取access_token以及openid的链接
public String getAccessToken(String code) {
    StringBuffer stringBuffer = new StringBuffer(getAccessTokenUrl());
    stringBuffer.append("appid=").append(appID);
    stringBuffer.append("&secret=").append(secret);
    stringBuffer.append("&code=").append(code);
    stringBuffer.append("&grant_type=authorization_code");
    return stringBuffer.toString();
}
    • controller中创建获取access_token以及openid的接口
@RequestMapping("/getUserInfo")
public void getUserInfo(@RequestParam String code) throws IOException {
    //构建获取 access_token链接
    String accessToken = wxConfigure.getAccessToken(code);
    //获取 access_token，返回的是一个json，token包含其中
    //该方法的作用相当于在服务端发送了一个get请求
    String callBackStr = HttpClientUtils.doGet(accessToken);
    JSONObject parse = JSONObject.parseObject(callBackStr);
    String access_token = parse.getString("access_token");
    String openid = parse.getString("openid");
}
    • 正确时返回的JSON数据包如下：
{
  "access_token":"ACCESS_TOKEN",
  "expires_in":7200,
  "refresh_token":"REFRESH_TOKEN",
  "openid":"OPENID",
  "scope":"SCOPE"
}
  • 3.通过access_token与openid获取用户详细信息
    • 参考以下链接构建请求链接： https://api.weixin.qq.com/sns/userinfo?access_token=ACCESS_TOKEN&openid=OPENID&lang=zh_CN
    • 链接构成：
      • access_token
      • openid
      • lang：返回国家地区语言版本，zh_CN 简体，zh_TW 繁体，en 英语
    • 将固定部分写入application.properties
wx.login.userInfoUrl=https://api.weixin.qq.com/sns/userinfo?
wx.login.lang=zh_CN
    • 创建构建请求链接方法
public String getUserInfo(String access_token,String openid) {
    StringBuffer stringBuffer = new StringBuffer(getUserInfoUrl());
    stringBuffer.append("access_token=" + access_token);
    stringBuffer.append("&openid=" + openid);
    stringBuffer.append("&lang="+lang);
    return stringBuffer.toString();
}
    • 在controller中添加获取userInfo的方法
//获取用户信息
String userInfoUrl = wxConfigure.getUserInfo(access_token, openid);
String userInfoStr = HttpClientUtils.doGet(userInfoUrl);
JSONObject userInfo = JSONObject.parseObject(userInfoStr);
log.info(userInfo.getString("nickname"));
    • 正确时返回的JSON数据包如下：
{   
  "openid":" OPENID",
  "nickname": NICKNAME,
  "sex":"1",
  "province":"PROVINCE",
  "city":"CITY",
  "country":"COUNTRY",
  "headimgurl":       "http://thirdwx.qlogo.cn/mmopen/g3MonUZtNHkdmzicIlibx6iaFqAc56vxLSUfpb6n5WKSYVY0ChQKkiaJSgQ1dZuTOgvLLrhJbERQQ4eMsv84eavHiaiceqxibJxCfHe/46",
  "privilege":[ "PRIVILEGE1" "PRIVILEGE2"     ],

}


## 附加
  • 刷新token
    • 获取第二步的refresh_token后，请求以下链接获取access_token： https://api.weixin.qq.com/sns/oauth2/refresh_token?appid=APPID&grant_type=refresh_token&refresh_token=REFRESH_TOKEN
  • 检验授权凭证（access_token）是否有效
    • http：GET（请使用https协议） https://api.weixin.qq.com/sns/auth?access_token=ACCESS_TOKEN&openid=OPENID

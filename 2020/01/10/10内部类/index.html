10内部类

内部类可以无条件访问外部类成员

//即使外部类中的域或者方法被指定为private

内部类创建必须要一个外部类对象

//除非这个内部类为套嵌类（静态内部类）

//普通的内部类不允许有static的数据和字段

 

内部类创建的位置是非常的随意的，它可以在类作用域、方法、甚至一个判断语句中：

class Out{

void one（）{

class In1{}

}

void two（）{

if（flag）

{

class In2{}

}

}

}

//除了作用域在方法中外，内部类和外部类性质相同

 

获取内部类对象：

//在获取外部类对象前无法获取内部类对象（除非是一个嵌套类）

使用.new关键字

class Out{

class In{}

main（）{

Out out=new Out();

Out.In in=out.new In();

//创建内部类引用时，使用的是：外部类.内部类

//创建引用和对象的声明形式不一样，这有点奇怪

}

}

 

内部类生成外部类引用：

使用.this关键字

class Out{

class In{

public Out get(){

return Out.this;

}

}

}

 

 

向上转型：

内部类的基类并非是外部类

被定义为private的内部类和接口只能在外部类的内部方法可以访问

//这意味着不能够使用new创建对象或者向上转型

//假如一个被定义为private的方法实现了某个接口，接口的引用不能接受来自该内部类的对象

 

匿名内部类：

//解释：创建实例的同时临时定义一个类

//匿名类一定是用于临时实现一个接口或者继承一个类

例：

Runnable x=new Runnable（）{

//Runnable是一个接口，run是这个接口中的方法

@Override

public void run（）{

print（“hello”）；

}

}

此过程涉及了向上转型//匿名类-->Runnable

更清楚的例子：

public Runnable gets（）{

return new Runnable（）{

@Override

public void run（）{}

}

}

//匿名类没有命名构造器（它没有名字）

//匿名类和类一样拥有实例初始化方法(此方法相当于构造函数…)

//匿名类可以携带参数来实现基类的构造器（基类如果只有一个有参构造函数的话）

//实现基类构造函数是自动进行的（这意味着不允许主动调用super实现构造）

//匿名类中不允许使用static关键字

匿名类想要使用不完全闭包的对象，该对象的引用必须被指定为final！

//传递给基类的参数不需要被定义为final

int c=0;

public Runnable test（final int i）{

     return new Runnable（）{

           private int j=i；

           @Override

           public void run（）{

c++;

//类对方法构成完全闭包，所以匿名类可以自由调用外部类的自由变量c

//而参数i为不完全闭包关系，所以需要指定它的引用为final

//jdk1.8以后（也许是1.7）不完全闭包的变量隐式地被指定为final，不需要显式声明

}

     }

}

//更一般的情况详见闭包

 

嵌套类

嵌套类切断了与外部类对象（不是外部类）的联系

//普通内部类对象隐式保留了对外部类对象的引用

嵌套类被是被指定为static的内部类

//创建嵌套类对象不需要外部类对象（new Apple.Banana（）这种写法就可以）

//嵌套类不可以访问非static的外部类对象

嵌套类因为本身static的属性，所以它也可以出现在接口中，甚至直接实现接口

interface Apple{

 

public void print();

//测试嵌套在接口中的类

public static class Banana implements Apple{

//该嵌套类甚至直接实现了外围的接口        

public void print() {

System.out.println("hello!");

}

public static void main(String[] args) {

new Banana().print();

}

}

}

 

 

 

为什么需要内部类？

  1. 实现多重继承

java不允许类的多重继承（接口可以），但是内部类可以独立继承某个类（或者接口），因此内部类实现了多重继承

  1. 内部类无条件访问外部类的所有内容
  2. 内部类对接口实现具有独立性，单个外部类可以通过多个内部类对同一个接口多次实现

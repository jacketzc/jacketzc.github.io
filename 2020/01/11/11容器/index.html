11容器


java从根本上有两种容器（都是接口）
1.Collection：一个独立元素的序列，这些元素服从一条或者多条规则
//AbstractCollection为该接口的默认实现
//Iterable接口为所有序列接口的父接口
//这意味着Iterable接口会被强制实现
//Collection主要分为“List”和“Set”两类
2.Map：一组成对的“键值对”对象，允许使用键来查找值
//AbstractMap为该接口的默认实现
 
有必要了解：
带hash的容器将优化查找速度
带linked的容器将优化中途插入、删除的功能
java中的容器存放的都是对象引用，不能使用基本类型作为泛型（内存位置不同）
 
打印容器：
toString();方法将对每一个容器成员使用该方法，结果返回一个所有元素的字符串
//直接打印容器对象会默认调用该方法
//若想输出类信息以及地址，可以使用super.toString()（Object中的toString方法）
 
注意区分Iterator（迭代器）接口和Iterable（迭代）接口，后者才具有forEach方法并且返回一个Iterator
//如果要实现一个轻量级的消费Collection，可以直接实现一个Iterator接口即可，此方法的耦合度最低
 
List接口：
extends Collection
List承诺可以将元素维护在特定的序列中（Collection并不具备对List中间元素管理的功能）
//List允许元素的中间插入与删除
有两种基本的List：
1.ArrayList：擅长随机访问，中途插入或删除元素比较慢
//ArrayList基于数组实现，同时也实现了RandonAccess接口，使用for遍历比使用迭代器更为迅速
常用方法：
Arrays.asList（T…elements）
将一个数组转化为list，此方法可以快速构建一个list
//此list长度固定
bool addAll（Collection）
将一个集合插入到list中
可选：可指定插入位置
List subList（int，int）
获取list中指定区域的视图
//是视图，不是新的list
注：第一个参数标识起始位置，第二个标识结束位置
//包括起始位置，不包括结束位置
int indexOf（Object）
返回元素第一次出现的位置
void clear（）
删除所有元素
使用subList（）.clear（）可以从list中删除指定区域的元素
bool retainAll（Collection）
保留指定元素
//保留的元素为list与collection的交集
//该方法基于equals（）
bool removeAll（Collection）
删除指定元素
//删除也是与Collection的交集
//该方法基于equals（）
set （int， E）
//将指定位置的元素替换
2.LinkedList：以较低的代价在List中间插入或删除，优化了顺序访问，随机访问比较慢
//原因在于ArrayList底层使用数组实现的，而LinkedList使用链表实现的
//LinkedList的get（i）方法永远是从头开始寻找的
LinkedList不光实现了list接口，还实现了Queue和Deque接口（双端队列）
LinkedList具有实现栈的全部方法：
//所谓栈就是“先进后出”
addFirst（）：在栈首添加一个元素
getFirst（）：获取栈首元素
removeFirst（）：移除栈首元素
isEmpty（）：是否为空
contains（obj）：是否存在元素
//事实上，java中的Stack（栈）类就是使用LinkedList实现的
//对应方法为添加：push（）、移除：pop（）、获取栈首：peek（）、empty（）、    
搜索：search（）
 
 
Set接口：
set不允许有重复的元素
//set的另一个特点是自维护元素的存取顺序（存的顺序和取的顺序不一致）
//这就导致不可以使用和序号有关的方法，如get、set
set最常被用于测试归属性（是否含有某元素），因此通常使用HashSet来作为实现的类
//HashSet根据元素的hash值决定元素的顺序
//测试归属性使用了contains（）方法
//HashSet优化了快速查找
//TreeSet在存储的时候实现了排序
该接口实际的应用：
  1. 查找不相同元素的总数
 
 
Map接口：
//Map没有实现Iterable接口，没法遍历
map为key-value结构
常用方法：
put（key，value）
//key和value都是由任意的对象构成
//此处可以构成多维关系，一个key可以对应一个list引用
containsKey（）：根据key查询归属性
containsValue（）：根据value查询归属性
get（key）：根据key来获取value对象
//HashMap的键可以为null、TreeMap不可以
 
Queue接口：
队列是一个FIFO容器（先进先出）
//从容器的一段放入，另一端取出，放入与取出的顺序是相同的
//队列在并发编程中很重要，他可以安全地将一个对象传送给另一个任务
 
LinkList实现了该接口
常用方法：
add（E）：将指定元素插入到此队列中
remove（）：检索并删除该队列的头（空抛出异常）
element（）：检索该队列的头（不删除）//空队列抛出异常
offer（E）：将指定元素插入到队列中
peek（）：检索该队列的头（不删除）//空队列返回null
poll（）：检索并删除队列的头（空返回null）
//它的返回结果是该队列的头
//所有remove（）方法中，除了参数是对象的情况，其他重载方法返回的都是对象
 
PriorityQueue类实现了Queue接口
优先队列表明弹出的时候弹出的是优先级最高的元素
//优先队列在装入时已经按照某种规则对元素进行了排序（比如int类型从小到大排序）
//空格也可以作为String元素被填入，并且优先级高于字母
//该队列允许重复元素
//在构造方法中，可以指定排序规则（如Collections.reserveOrder（），相反顺序）
 
for each方法：
该方法由Iterable实现，因此所有的Collection的引用都可以使用
//for each方法的底层实现来源于Iterator
Map并没有对此方法的直接实现，因此需要基于Map返回一个集合视图（其实返回的是一个Set）：
for（Map.Entry<T,T> entry : map.entrySet()）
syso(entry.getKey()+entry.getValue());
//Map.Entry是一个单独的接口，获取引用的唯一方法为Map.entrySet（）
如果只需要遍历key，可以直接获取一个key视图：
for（T element：map.keySet()）
syso(element）；
如果需要遍历value，可以获取value视图：
for（T element：map.value（））
syso（element）
//数组不属于Iterable，但是仍然可以用forEach，此处foreach语法在编译后转变成了一个普通的for循环
 
自定义Iterable接口实现for each遍历：
//实现Iterable接口的类的对象就可以使用forEach遍历
Iterable接口中只有唯一可实现的方法：iterator（），该方法返回一个迭代器
例：
class Apple<T> extends ArrayList<T>{
//返回一个Iterator接口引用的方法，可以使用匿名类实现
//匿名类中可以自定义Iterator的属性，因此也实现 了for each的另类实现方法（倒叙输出）
public Iterable reversed（）{
//只需调用.reversed()就可以返回一个可供遍历的Iterable引用
return new Iterable（）{

return new Iterator（）{
int current=size()-1;
public boolean hasNext(){return current>-1;}
public T next(){return get(current--)}
public void remove(){}
//因为只需要遍历，所以remove（）方法可以不实现
}
}
}
}
}

Collections类：
//此类仅由静态方法组成或返回集合，它包含对集合进行操作的多态方法
常用方法：
static <T> bool addAll（Collection，T…elements）
//添加所有元素
staitc <T> List<T> unmodifiableList（）
//返回一个不可修改的list（传递给final的List引用）

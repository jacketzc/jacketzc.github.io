2.0Aspectj自动AOP注入

AOP：面向切面编程
说人话就是在核心业务代码前后添加一些业务处理，然后把这些业务处理分离

导入新的依赖
<dependency>
  <groupId>org.aspectj</groupId>
  <artifactId>aspectjweaver</artifactId>
  <version>1.9.5</version>
</dependency>
//下方依赖已经包括了cglib的依赖
<dependency>
  <groupId>org.springframework</groupId>
  <artifactId>spring-aop</artifactId>
  <version>${spring_version}</version>
</dependency>


@Before：前置通知，在方法执行之前执行
@After：后置通知，在方法执行后执行
//不论方法有没有执行成功，都会执行通知
@AfterRunning：方法返回后通知，可以获取到返回值
@AfterThrowing：异常抛出通知，方法抛出后通知，可以捕获到异常
@Around：环绕通知，以上四种通知的结合体

操作：
#bean.xml
  • 开启组件扫描
  • 开启aspectj自动注解
    • 默认情况下使用jdk代理，设置proxy-target-class属性为true时将使用cglib代理
<context:component-scan base-package="com.jac.calculator"/>
//如果使用jdk代理，那么获取bean的时候只能获取目标类的接口.class
<aop:aspectj-autoproxy/>
//<aop:aspectj-autoproxy proxy-target-class="true"/>
#需要增强的类（也就是一个需要添加业务的普通类）
@Component("arithmetic")
public class  ArithmeticCalculatorImpl implements ArithmeticCalculator{
    @Override
    public int add(int x, int y) {
        return x + y;
    }
    @Override
    public int sub(int x, int y) {
        return x - y;
    }
    @Override
    public int mul(int x, int y) {
        return x * y;
    }
    @Override
    public int div(int x, int y) {
        return x / y;
    }
}
#添加一个切面类
  • 添加@Aspect表明该类为增强类
  • 添加@Order指定该增强类的优先级
  • 使用@Pointcut标注一个共用的切入点（切入点就是被增强的类）
  • 可以在通知（切入方法）中指定joinpoint参数，来获取AOP的相关信息：
    • 连接点（就是被增强的方法）的相关信息
//（execution(int com.jac.calculator.ArithmeticCalculator.add(int,int))）
    • AOP代理对象
    • 目标对象
    • 被通知方法参数类别等
  • 在环绕通知中必须指定processjoinpoint来手动放行方法
  • 返回通知以及异常通知必须在注解中指定相应的属性才能获取到返回值或者异常
@Aspect//标识成一个切面
@Component//表示为手spring管理的组件
@Order(1)//如果对相同的类进行增强，可以指定优先级，数字越小优先级越高
public class LoginAspectj {
//    切入点表达式可以复用，如果该表达式在其他的类（同一个包）中，可以使用“LoginAspectj.pointcut()”来指定
//    如果不在同一个包中，需要额外指定类名
    @Pointcut("execution(* com.jac.calculator.*.*(..))")
    public void pointcut() {
    }
//    @Before("execution(* com.jac.calculator.ArithmeticCalculatorImpl.add(..))")
    @Before("pointcut()")
    public void beforeMethod() {
        System.out.println("before:");
    }
    @After("pointcut()")
    public void afterMethod(JoinPoint joinPoint) {
//        获取目标方法名
        String name = joinPoint.getSignature().getName();
        System.out.println("After,执行的方法:"+name);
    }
//  如果需要获取返回值，注解中的returning参数需要和方法参数一致
    @AfterReturning(value = "pointcut()",returning = "res")
    public void afterReturn(JoinPoint joinPoint,Object res) {
        System.out.println("AfterRunning,返回值:"+res);
    }
//    如果需要捕获异常，注解中throwing需要和方法参数一致
//    可以通过形参指定异常类型，设置固定异常类型才会执行异常通知
    @AfterThrowing(value = "pointcut()",throwing = "e")
    public void afterThrow(JoinPoint joinPoint,Exception e) {
        System.out.println(e);
    }
    @Around("pointcut()")
    public Object aroundMethod(ProceedingJoinPoint joinPoint) {
        try {
            System.out.println("around-before:");
//            前置通知处理代码（相当于前置通知）
            return joinPoint.proceed();
        }catch (Throwable e){
//            异常通知执行代码（相当于异常通知）
            System.out.println("around-"+e);
        }finally {
//            后置通知执行代码
            System.out.println("around-after:");
        }
        return null;
    }
}

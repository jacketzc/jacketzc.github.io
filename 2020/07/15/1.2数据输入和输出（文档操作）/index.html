1.2数据输入和输出（文档操作）

#文档
指最顶层或者根对象, 这个根对象被序列化成 JSON 并存储到 Elasticsearch 中，指定了唯一 ID
//相当于对象（hash、map、JSON），这些对象中可以包含另一些对象
//Elasticsearch不允许字段的名字（key）包含英文句号(.)


#文档元数据
文档不仅仅包含数据，也包含元数据（有关文档的信息），包括
  • _index：文档的存放位置（相当于一个数据库）
一个索引应该是因共同特性被分组到一起的文档集合（参考数据库设计范式 07数据库设计）
可以让Elasticsearch帮我们管理索引，但是有一些创建时的要求：
    • 必须小写
    • 不能以下划线开头
    • 不能包含逗号
  • _type：文档表示的对象类型（相当于一个表）
  • _id：文档唯一标识
//通过_index、_type和_id的组合可以确定一个文档
  • _version：文档版本号，每次对文档进行修改时，版本号对递增，用于冲突处理

#使用自定义ID
PUT /{index}/{type}/{id}{
  "field": "value",
  ...
}
例：
PUT /website/blog/123{
  "title": "My first blog entry",
  "text":  "Just trying this out...",
  "date":  "2014/01/01"
}
响应：
{
   "_index":    "website",
   "_type":     "blog",
   "_id":       "123",
   "_version":  1,
   "created":   true
}

#自动生成id
将PUT请求修改为POST，整个URL只需要包含_index以及_type
POST /website/blog/{
  "title": "My second blog entry",
  "text":  "Still trying this out...",
  "date":  "2014/01/01"
}
返回：
{
   "_index":    "website",
   "_type":     "blog",
   "_id":       "AVFgSgVHUP18jI2wRx0w",
   "_version":  1,
   "created":   true
}


#取出一个文档
使用GET请求来检索文档
  • 请求参数中添加pretty参数可以使得JSON响应更加可读
//传入的JSON数据将会被原样输出
GET /website/blog/123?pretty
响应：
  • found参数证实了文档已被找到
{
  "_index" :   "website",
  "_type" :    "blog",
  "_id" :      "123",
  "_version" : 1,
  "found" :    true,
  "_source" :  {
      "title": "My first blog entry",
      "text":  "Just trying this out...",
      "date":  "2014/01/01"
  }
}
  • 返回部分文档
可以在url中指定_source参数来指定最终获取到的_source中出现的内容
GET /website/blog/123?_source=title,text
  • 只返回_source字段
GET /website/blog/123/_source
#一次性取出多个文档
使用_mget查询参数
GET /_mget
{
   "docs" : [
      {
         "_index" : "website",
         "_type" :  "blog",
         "_id" :    2
      },
      {
         "_index" : "website",
         "_type" :  "pageviews",
         "_id" :    1,
         "_source": "views"
      }
   ]
}



#检验文档是否存在
使用HEAD请求，该请求的响应只有一个HTTP请求报头，如果文档存在，Elasticssearch将返回一个200 ok的状态码
HTTP/1.1 200 OK
Content-Type: text/plain; charset=UTF-8
Content-Length: 0


#更新文档
  • PUT "/website/blog/123"
//将之前的文档替换为提交的内容
PUT /website/blog/123{
  "title": "My first blog entry",
  "text":  "I am starting to get the hang of this...",
  "date":  "2014/01/02"
}
  • POST "/website/blog/123"
//在原有的文档上添加属性
POST /website/blog/1/_update
{
   "doc" : {
      "tags" : [ "testing" ],
      "views": 0
   }
}
Elasticsearch中的文档是不可改变的，当我们修改文档时，其实是新建了一个文档
新文档的_version会加一
流程：
  • 从旧文档构建JSON
  • 更改该JSON
  • 删除旧文档（标记删除、其实还会保存一段时间）
  • 索引新文档

#删除文档
DELETE "/website/blog/123"
//删除时，会将文档标记为删除状态（不会立即删除），同时该文档的version+1
//如果没有找到该文档，该请求所期待的文档version仍然会+1
{
  "found" :    false,
  "_index" :   "website",
  "_type" :    "blog",
  "_id" :      "123",
  "_version" : 4
}
#删除所有文档
POST /**/**/_delete_by_query
{
    "query":{
        "match_all":{}
    }
}

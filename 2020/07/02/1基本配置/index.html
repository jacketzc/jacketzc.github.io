1基本配置

# 默认配置：
  • log4j2默认会在classpath目录下寻找log4j.json、log4j.jsn、log4j2.xml等名称的文件，如果都没有找到，则会按默认配置输出，也就是输出到控制台
<?xml version="1.0" encoding="UTF-8"?>
<Configuration status="WARN">
    <Appenders>
        <Console name="Console" target="SYSTEM_OUT">
            <PatternLayout pattern="%d{HH:mm:ss.SSS} [%t] %-5level %logger{36} - %msg%n" />
        </Console>
    </Appenders>
    <Loggers>
        <logger name="mylog" level="debug" additivity="false">
            <AppenderRef ref="Console"/>
        </logger>
        <Root level="error">
            <AppenderRef ref="Console" />
        </Root>
    </Loggers>
</Configuration>




<Configuration status=“warn”monitorInterval="300">
//配置的根节点
  •  status属性规定了log4j2自己产生的日志的输出级别，日志级别从低到高分为TRACE < DEBUG < INFO < WARN < ERROR < FATAL，如果设置为WARN，则低于WARN的信息都不会输出。对于Loggers中level的定义同样适用
  • monitorInterval属性指定了log4j2重新读取配置文件的间隔

<Appenders>
//配置输出方式
  • 默认配置下，设置了一个<Console>表明向控制台输出，其中的<PatternLayout>定义了输出日志的格式

<loggers>
//日志配置
  • 默认配置了一个<Root>的logger，表明对任何的日志都生效，level属性标注了该配置下的输出等级，<AppenderRef>定义了该配置使用的输出方式
//如果自定义输出级别低于warn，则会使用warn代替，比warn高则使用自定义的输出级别
  • 可以使用<logger>标签自定义logger，需要指定name（该logger的名称），level（输出等级）
    //添加additivity="false"属性，表示在该配置下输出的日志不会在父级配置中再输出
    //如果该logger的name为某个类的全类名，那么该日志配置将自动指向那个类，如果该name为包全名，那么日志配置将指向该包中的所有类

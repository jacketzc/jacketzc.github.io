1.9文件下载和上传

#文件下载
借助HttpMessageConveter实现
下载原理：将发布到服务器上的文件，以流的方式写入到客户端
ResponseEntity：可以将下载的文件的数据及相应信息进行封装，浏览器解析该对象，实现下载
  • ServletContext：这个对象可以获取web应用上下文的对象（没法读取resources目录也就是项目上下文中的文件）
//普通的InputStream比如XXX.class.getClassLoader().getResourceAsStream()只能读取项目上下文的文件，读取不到web应用上下文的文件
@RequestMapping("imageDownload")
    public ResponseEntity<byte[]> imageDownload(HttpSession session) throws IOException {
//        定义一个图片的缓存字节数组
        byte[] images;
//        借助servlet的上下文context对象进行获取输入流
        ServletContext context = session.getServletContext();
//        读取图片输入流对象
        InputStream is = context.getResourceAsStream("images/006.jpg");
//        定义初始化的图片缓存数组大小，图片有多大就定多大
        images = new byte[is.available()];
//        将图片内容全部读取到字节数组中
        is.read(images);
//        处理响应数据，包括响应头信息处理，及封装到ResponseEntity对象中
//        ResponseEntity支持三个参数：1.发送给浏览器的数组 2.响应头信息 3.响应码
        HttpHeaders headers = new HttpHeaders();
//        attachment代表福建 filename 表示下载文件的名称（不是源文件的名称）
        headers.add("Content-Disposition", "attachment;filename=006.jpeg");
        //    设置响应状态码      Content-Disposition
        HttpStatus status = HttpStatus.OK;
//        将上述三个对象封装到响应实体中
        ResponseEntity<byte[]> responseEntity = new ResponseEntity<>(images, headers, status);
        return responseEntity;
    }

#文件上传
需要的依赖：
<dependency>
  <groupId>commons-io</groupId>
  <artifactId>commons-io</artifactId>
  <version>2.4</version>
</dependency>
<dependency>
  <groupId>commons-fileupload</groupId>
  <artifactId>commons-fileupload</artifactId>
  <version>1.3.1</version>
</dependency>
  • 前端
    • 使用H5的表单上传
<form action="fileUpload" method="post" enctype="multipart/form-data">
    上传文件：<input type="file" name="uploadFile">
    文件描述：<input type="text" name="uploadDesc">
    <input type="submit" value="上传单个图片">
</form>
<form action="fileUploads" method="post" enctype="multipart/form-data">
    <%--    多个文件上传，给file表单元素增加一个属性--%>
    上传文件：<input type="file" name="uploadFile" multiple="multiple">
    文件描述：<input type="text" name="uploadDesc">
    <input type="submit" value="上传单个图片">
</form>
    • 使用ajax上传（此处使用axios的ajax）
//指定一个表单元素（调用方法时并没有指定参数）
<input type="file" @change="sendImg" accept="image/jpg,image/jpeg,image/png"
       name="file" id="inputImage" class="hidden">
//js部分：axios发送文件时不需要指定content-type为multipart/form-data
sendImg(source) {
//    发送图片
    let file = source.target.files[0]
    let url = '/STU_SSM/fileUpload'
    let param = new FormData()
//param是一个表单对象，虽然往该对象中存放文件，但是仍然是键值对的形式
//此处的'file'相当于指定了表单元素的name，后台的@RequestParam需要指定该值
    param.append('file',file)
    console.log(param.get('file'))
    axios.post(url,param).then(function (res) {
        console.log(res.data)
    })
},
  • 配置文件上传解析器
<!--    配置文件上传解析器
        id必须是“multipartResolver”
        否则SpringMVC前端处理器初始化时找不到该id对应的实体（id值是定死的），就会报错
-->
    <bean id="multipartResolver" class="org.springframework.web.multipart.commons.CommonsMultipartResolver">
<!--        保证必须与上传表单的编码一致-->
        <property name="defaultEncoding" value="utf-8"/>
<!--        配置上传文件的最大支持容量大小，单位为字节（1M=1024*1024）-->
        <property name="maxUploadSize" value="5242880"/>
    </bean>
  • 配置控制器：
    @ResponseBody
    @RequestMapping("fileUpload")
    public void fileupload(@RequestParam("uploadFile")MultipartFile multipartFile,@RequestParam(value = "uploadDesc",required = false)String fileDesc,HttpSession session) throws IOException {
//        获取上传文件的名称
        String filename = multipartFile.getOriginalFilename();
//        获取上传文件的服务器的真实绝对路径（不是项目中的路径哦）
        ServletContext context = session.getServletContext();
        String realPath = context.getRealPath("uploads");
//        log.info("fileDesc:"+fileDesc);
        log.info("上传文件的真实绝对路径：" + realPath);
//        创建上传目标文件对象
        File targetFile = new File(realPath + "/" + filename);
//        执行上传操作
        multipartFile.transferTo(targetFile);
    }
  • 多文件上传
    @ResponseBody
    @RequestMapping("fileUploads")
    public void fileUploads(@RequestParam("uploadFile")MultipartFile[] multipartFiles,@RequestParam(value = "uploadDesc",required = false)String fileDesc,HttpSession session) throws IOException {
//        获取上传文件的服务器的真实绝对路径（不是项目中的路径哦）
        ServletContext context = session.getServletContext();
        String realPath = context.getRealPath("/")+"uploads";
        log.info("上传文件的真实绝对路径：" + realPath);
        for (MultipartFile multipartFile : multipartFiles) {
            String filename = multipartFile.getOriginalFilename();
            //        创建上传目标文件对象
            File targetFile = new File(realPath + "/" + filename);
            //        执行上传操作
            multipartFile.transferTo(targetFile);
        }
    }

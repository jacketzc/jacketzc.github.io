2.1Git本地高级操作

git reflog
//此方式能看到所有的commit、回退记录 

git reset 用法：
1.git reset --soft 版本号
将HEAD引用指向给定提交。索引（暂存区）和工作目录的内容是不变的
//每次commit都会生成一个版本号（一般是开头的7位，版本号不一定需要完整输入，输入4-5位足够软件判断即可）
2.git reset --mixed（默认） 版本号
HEAD引用指向给定提交，并且索引（暂存区）内容也跟着改变，工作目录内容不变。
3.git reset --hard 版本号
回退版本，HEAD引用指向给定提交，索引（暂存区）内容和工作目录内容都会变给定提交时的状态。

git revert用法
相当于先commit一次，再使用git reset --hard将版本恢复到上个版本


暂存：
git stash
还原暂存（使用上一次的暂存）：
git stash pop
//还原暂存相当于是add后的状态
清空暂存
git stash clear

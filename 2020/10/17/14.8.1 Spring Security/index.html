14.8.1 Spring Security

## 说明
>    spring提供的安全框架
>    通常情况下，要限制对web应用某些页面的访问（后台要登录等），可以使用过滤器、拦截器来实现，spring提供了基于Spring AOP与servlet过滤器的安全框架

## 入门（使用内存中保存的用户以及密码）
>    依赖
  • 添加过依赖后，再访问原来应用的页面就会提示需要用户名和密码
  • 默认的用户名是“user”，密码使用uuid生成会输出到日志中
<dependency>
     <groupId>org.springframework.boot</groupId>
     <artifactId>spring-boot-starter-security</artifactId>
</dependency>
>    配置
  • @EnableWebSecurity将开启Spring Security功能
  • @EnableGlobalMethodSecurity(prePostEnabled = true)将开启security注解
    • 可以在需要权限控制的方法上使用@PreAuthorize、@PreFilter这些注解
  • 继承WebSecurityConfigurerAdapter并重写其中的一些方法来实现自定义配置
  • 一共就两个步骤
    • 设置认证
    • 设置用户角色
//这个注解自带@Configuration
@EnableWebSecurity
@EnableGlobalMethodSecurity(prePostEnabled = true, securedEnabled = true, jsr250Enabled = true)
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    //授权
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        //首页所有人可以访问，功能页只有对应有权限的人才能访问
        //请求授权的规则~
        http.authorizeRequests()
                .antMatchers("/").permitAll()
                .antMatchers("/level1/**").hasRole("vip1")
                .antMatchers("/level2/**").hasRole("vip2")
                .antMatchers("/level3/**").hasRole("vip3");
        //没有权限默认跳转到登录页，现在开启登录的页面
        //默认跳转到/login，可以自定义login的页面
        //可以自定义登录登录认证的地址
        //可以自定义失败的地址
        //默认情况下，要求的表单输入框的 name为 username和 password
//        http.formLogin();
        http.formLogin().loginPage("/toLogin").loginProcessingUrl("/loginVerify");


        //开启注销功能
        //默认跳转到/login?logout，可以自定义注销成功后的位置
        http.logout().logoutSuccessUrl("/");
        //开启预防csrf攻击可能会自定义登入、登出失败，可以关闭这个功能
        http.csrf().disable();
        //开启记住我功能（cookie）
        //可以自定义前端发送请求的 name
        http.rememberMe().rememberMeParameter("remember");
    }
    //认证（使用内存中的账户）
    //在spring security 5.0中，不允许存入明文密码，需要指定加密方式进行加密
    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        String encode = new BCryptPasswordEncoder().encode("12345");
        auth.inMemoryAuthentication().passwordEncoder(new BCryptPasswordEncoder())
                .withUser("jacketzc")
                .password(encode)
                .roles("vip2", "vip3")
                .and()
                .withUser("root")
                .password(encode)
                .roles("vip1", "vip2", "vip3")
                .and()
                .withUser("guest")
                .password(encode)
                .roles("vip1");
    }
>    基于数据库的认证：
  • 将用户的信息保存在数据库中，需要的信息有
    • username、password、role（角色信息）
    • 可以建两个表
  • 创建类实现UserDetailsService，从数据库加载用户信息
    • 这个UserSecurityService是自己创建的，它要求根据传进来的username来获取到用户的password、role
@Component
public class SecurityUserConfig implements UserDetailsService {
    @Autowired
    private UserSecurityService service;
    @Autowired
    private PasswordEncoder passwordEncoder;

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        UserInfoCron userInfo = service.queryUserByName(username);
        Assert.notNull(userInfo,"用户不存在");
        Assert.isTrue(userInfo.getEnabled() == 1, "账户不可用");
        //角色集合
        List<GrantedAuthority> authorities = new ArrayList<>();
        userInfo.getAuthority().forEach(role -> authorities.add(new SimpleGrantedAuthority("ROLE_" + role)));

        return new User(username, passwordEncoder.encode(userInfo.getPassword()), authorities);
    }
}

## 角色权限控制
>    将某个接口只开放给某个用户
>    Spring Security提供了Spring EL表达式，允许我们在定义URL路径访问(@RequestMapping)的方法上面添加注解，来控制访问权限
public class TestServiceImpl {
    //该接口只开放给拥有ADMIN角色的账户
    //这个hasRole('ADMIN')是一个EL表达式，表示调用hasRole方法后的返回值，返回true的集合访问
    @PreAuthorize("hasRole('ADMIN')")
    @GetMapping("/hello")
    public String hello() {
        return "hello";
    }
    //该接口开放给拥有USER角色的账户
    @PreAuthorize("hasRole('USER')")
    @PostMapping("/pojo")
    public DefaultCons pojo(@RequestBody DefaultCons cons) {
        return cons;
    }
}
